# Specification - Implement Backup Retention Rules

## 1. Overview
The goal is to implement retention policies for backups generated by `docker-backup.sh`. When creating a new backup, the script should automatically prune older backups based on a retention configuration to prevent disk exhaustion.

## 2. Requirements
- **Pruning Strategy**: Implement a simple retention scheme, such as keeping the last $N$ backups (e.g., keep 5 backups).
- **Command Line Integration**: 
  - Add a flag or option to set retention limit, e.g., `--keep <num>` or `-keep <num>` (or use a default value if not specified, e.g., default 5 backups).
- **Safety**: Only delete files matching the application's backup naming convention (e.g., `<app_name>_backup_<timestamp>.tar.gz`). Never delete other files.
- **Logging**: Log all pruning activities, including which backups are retained and which are deleted.

## 3. Testing Plan
- Test script behavior using a test harness in Bash.
- Simulate backup generation and verify retention rules prune the correct files.
